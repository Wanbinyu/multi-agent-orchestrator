## ❌ 失败原因分析

Reviewer 反馈的问题可归为三类，均与**模型连接无关**：

| 问题 | 根因 |
|------|------|
| 正文严重偏离大纲 | `writer` Worker 只收到自己的任务描述，**未读取 `plot_designer` 生成的大纲**。 |
| 第二章输出为空 | `kimi-for-coding` 接收到的上下文不完整或任务输入不足。 |
| 各章节互相无关联 | 4 个 `writer` **并行独立执行**，彼此不知道其他章节内容。 |
| 一致性检查和润色未生效 | `continuity_checker`、`editor` 需要上游 Worker 的输出内容作为输入，但当前流程未传递。 |

---

## 🔧 暴露的系统缺口

对照 `docs/迭代计划.md`，本次运行验证了以下待改进点：

1. **任务依赖调度（步骤 3）**
   - 当前：所有任务一把并发。
   - 目标：支持 DAG 依赖，例如 `大纲 → 正文 → 检查 → 润色`。

2. **Worker 间上下文传递**
   - 当前：下游 Worker 看不到上游产出。
   - 目标：Orchestrator 在任务 `input` 中明确要求读取上游输出文件。

3. **`read_file` 工具未真正启用（步骤 4）**
   - 当前：`src/tools/worker_tools.py` 已实现该工具，但任务拆分时尚未要求模型使用。
   - 目标：让 `writer`、`editor`、`continuity_checker` 都能读取所需上下文文件。

## 📝 后续建议

| 优先级 | 动作 | 预期效果 |
|--------|------|----------|
| 高 | 让 Orchestrator 输出 `depends_on` 依赖关系 | 控制执行顺序 |
| 高 | 让 Dispatcher 按 DAG 调度，而非全并行 | 避免下游先跑、上游未跑完 |
| 高 | 在任务 input 中注入需读取的上游文件路径 | Worker 能拿到完整上下文 |
| 中 | 让 Reviewer 未通过时支持自动重试或人工干预 | 提升最终成功率 |
| 低 | 长文本创作场景先单模型端到端验证 | 降低多 Worker 协调复杂度 |
