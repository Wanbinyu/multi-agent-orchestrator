## 4. 新设备恢复步骤

```bash
git clone https://github.com/Wanbinyu/multi-agent-orchestrator.git
cd multi-agent-orchestrator
git status --short --branch
git pull --ff-only
python -m venv .venv
```

激活环境后：

```bash
python -m pip install -e ".[test]"
python -m pytest -q
mao --version
mao web --help
```

不要复制旧电脑的 `.env` 到公开位置。需要的 Provider Key 在新设备本地重新配置。

## 5. 每次开始工作的固定步骤

```bash
git status --short --branch
git fetch origin
git rev-list --left-right --count HEAD...origin/main
```

然后：

1. 阅读本文和当前版本执行清单。
2. 确认工作树中的改动来源，不回滚不属于当前任务的修改。
3. 只把一个步骤标记为进行中。
4. 修改前记录验收标准和针对性测试。
5. 修改后先跑针对性测试，再按风险扩大回归。

## 6. 常用运行命令

安装用户：

```bash

```

源码开发：

```bash

mao

```

一次性任务：

```bash
mao run "检查当前项目并给出风险"
```

## 8. 文档更新规则

每完成一个 Beta 子任务，同时更新：

- 本文的当前状态和第一任务。
- 对应版本执行清单的复选框、测试结果和剩余风险。
- 影响架构时更新 `MAO-架构概览.md`。
- 改变版本目标时更新版本计划，不在多个旧计划重复维护。
- 发布时更新 CHANGELOG 和不可变 Release Notes。

已完成的整阶段计划移入 `docs/archive/`，但当前执行清单在版本发布前不能归档。

## 9. 安全操作

- Key 只放本地 `.env`，不得写入 Markdown、Issue、日志或命令输出。
- 曾在对话中出现的 Key 应视为泄露并在 Provider 控制台轮换。
- 不运行来自未知仓库的安装脚本或插件。
- 不自动改 GitHub 可见性、Tag 或 Release；这些操作需要所有者单独确认。
- 不对用户已有 Git 改动执行 reset、checkout 或覆盖写入。
