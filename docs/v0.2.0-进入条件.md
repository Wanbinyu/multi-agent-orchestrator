## P0 审计结论

2026-07-21 对密钥处理、越界写入、任务假完成、插件/工具越权、命令执行五类做 P0 审计。

**发现 1 个 P0，已修复**：

- `run_command`（`src/tools/worker_tools.py`）允许 `python -c`/`node -e`/`node --eval`/`node -p`/`node --print` 解释器内联代码执行。前缀白名单（`"python "`、`"node "`）放行这些命令，且无内联代码标志检查（`frontend_smoke.py` 与 `benchmark.py` 已有同类检查但未接入 `run_command`）。模型经提示注入或幻觉发出 `python -c "import os; os.system(...)"` 即可读 `.env`（密钥泄露）、越界写（绕过 `permission_rules`）、网络请求或破坏数据；`auto` 模式下无确认执行。
- **修复**：`run_command` preflight 新增 `_has_inline_interpreter_code` 检查，对 `python -c`（且无 `-m`，避免误拒 `python -m pytest -c config`）与 `node -e/--eval/-p/--print` 返回 `inline_interpreter_code` 错误并拒绝执行。新增/更新测试覆盖拒绝、模块模式不误拒、脚本文件不误拒。
- **影响范围**：`v0.1.0-beta.6` 及更早的 `run_command` 均受影响；`readonly` 会话不受影响（`run_command` 类别为 `execute`，readonly 直接拒绝）。

**其余四类 SOUND**：密钥只存 `.env`、`providers.yaml` 仅存 `${VAR}` 引用、错误/诊断脱敏（仅异常类型）；文件工具 `resolve_path` 归属约束 + `permission_rules` deny>ask>allow + Worker 写隔离，Workers 无法经拆任务扩权；`CompletionAudit` 纯确定性，模型正文不能覆盖，`AdversarialTester`/`Reviewer` 不能把失败升级为完成；插件注册的工具经同一 `permission_rules.decide` 检查，不绕过权限层（Python 插件同进程权限是设计内非 P0）。
