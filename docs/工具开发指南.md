# MAO 工具开发指南

> 如何为 MAO 设计与添加新工具。面向第三方开发者。
> MAO 的工具系统基于 `ToolRegistry` 单例，工具在导入时自动注册，CLI/Web/原生 tool_use 三端自动可用。

---

## 一、核心概念

| 概念 | 说明 |
|---|---|
| `ToolRegistry` | 全局工具注册表单例 `tool_registry`，管理所有工具的元数据与执行 |
| `ToolSpec` | 工具元数据：name / description / params / callable / category |
| `ToolResult` | 工具返回值：`success: bool` + `output: str` + `error: str` |
| `category` | 工具分类：`read` / `write` / `execute` / `external` / `unsafe`（用于权限与展示） |

工具调用有两种模式，**开发者无需关心**，注册表自动适配：
- **Markdown 模式**（默认）：工具以 ```` ```tool:name ```` 代码块调用，兼容所有模型。
- **原生 tool_use 模式**：对声明 `tool_use` 能力的模型，自动生成原生 schema 经 `tools=` 传入。

---

## 二、添加一个本地工具（最常见）

### 步骤

1. 在 `src/tools/` 或 `src/tools/contrib/` 新建模块。
2. 用 `@tool_registry.register(...)` 装饰函数，返回 `ToolResult`。
3. 在 `src/tools/worker_tools.py` 顶部追加 `import` 语句（触发注册）。

### 完整示例

```python
# src/tools/contrib/my_tools.py
from __future__ import annotations
from src.tools.registry import tool_registry
from src.tools.tool_result import ToolResult


@tool_registry.register(
    name="word_count",
    description="统计文本的字符数、单词数、行数",
    params={
        "text": {"type": "string", "description": "要统计的文本"},
    },
    category="read",
)
def word_count(text: str, base_dir: str = ".") -> ToolResult:
    try:
        chars = len(text)
        words = len(text.split())
        lines = text.count("\n") + (1 if text.strip() else 0)
        return ToolResult(
            success=True,
            output=f"字符数：{chars}\n单词数：{words}\n行数：{lines}",
        )
    except Exception as e:
        return ToolResult(success=False, error=str(e))
```

### 注册参数说明

| 参数 | 必填 | 说明 |
|---|---|---|
| `name` | ✅ | 工具唯一标识，模型用它调用 |
| `description` | ✅ | 工具功能描述，会进系统提示/原生 schema |
| `params` | ✅ | 参数 JSON Schema 字典；每个 key 是参数名，value 是 schema 片段 |
| `category` | ✅ | `read`/`write`/`execute`/`external`/`unsafe` |

### params 写法

```python
params={
    "path": {"type": "string", "description": "文件路径"},           # 必填
    "top_k": {"type": "integer", "description": "数量", "default": 5},  # 可选（有 default）
}
```
- 有 `default` 的参数视为可选，不会进入原生 schema 的 `required`。
- `base_dir` / `allowed_prefixes` 是框架自动注入的特殊参数（见下），**不要**写进 params。

### 返回值

始终返回 `ToolResult`：
```python
ToolResult(success=True, output="结果文本")
ToolResult(success=False, error="错误原因")
```
- `output` 会作为工具结果回传给模型，应是人类/模型可读的文本。
- 永远不要抛异常给上层--在工具内 `try/except` 并返回 `success=False`。

## 三、特殊参数（框架自动注入）

如果你的工具函数签名包含以下参数，注册表执行时会自动注入，模型不会看到它们：

| 参数 | 注入值 | 用途 |
|---|---|---|
| `base_dir: str = "."` | 当前会话输出目录 | 解析相对路径、定位文件 |
| `allowed_prefixes: list[str] \| None = None` | 命令白名单 | `run_command` 类工具 |

路径解析请复用 `src.tools.paths.resolve_path`，它防止目录穿越：
```python
from src.tools.paths import resolve_path as _resolve_path

target = _resolve_path(path, base_dir)  # 相对路径限制在 base_dir 内
```

## 四、category 与权限

| category | 含义 | readonly 模式 | approve 模式 |
|---|---|---|---|
| `read` | 只读（读文件、搜索） | 拒绝（当前实现拒绝所有工具） | 逐次确认 |
| `write` | 写文件、编辑 | 拒绝 | 逐次确认 |
| `execute` | 执行命令 | 拒绝 | 逐次确认 |
| `external` | 网络调用（搜索、抓取） | 拒绝 | 逐次确认 |
| `unsafe` | 高风险（完整 shell 等） | 拒绝 | 逐次确认，建议仅 opt-in |

> 当前 `readonly` 模式拒绝所有工具调用（与历史行为一致）。`category` 主要用于展示与未来的细粒度权限。

## 五、添加外部工具源（MCP 等）

若工具来自外部服务器（如 MCP），实现 `ToolSource` 协议并注册：

```python
from src.tools.registry import tool_registry, ToolSpec
from src.tools.tool_result import ToolResult

class MyToolSource:
    def list_tools(self) -> list[ToolSpec]:
        # 从外部服务器拉取工具列表，转为 ToolSpec
        return [ToolSpec(name="ext_tool", description="...", params={...},
                         callable=lambda **_: ToolResult(success=True, output=""),
                         category="external")]
    def execute(self, name: str, params: dict) -> ToolResult:
        # 调用外部服务器执行
        return ToolResult(success=True, output="...")

tool_registry.add_source(MyToolSource())
```

注册后，外部工具自动纳入发现与执行链路，本地同名工具优先。

> MCP 适配器已在 `src/tools/mcp_adapter.py` 实现，支持 stdio / SSE；`src/tools/tool_sources.py` 保留为兼容导出入口。

## 六、注册后工具自动出现在

1. **Agent 系统提示**（Markdown 模式）：`tool_registry.build_instructions()` 自动列出。
2. **原生 tools= 参数**（原生模式）：`tool_registry.build_tool_schemas()` 自动生成 schema。
3. **CLI `/tools` 命令**：动态列出所有注册工具。
4. **Web 右侧边栏本轮记录**：工具调用自动展示。
5. **协作 Worker**：在 `config/workers.yaml` 的 `tools:` 列表加入工具名即可授予 Worker。

## 七、测试你的工具

参照 `tests/test_search_tools.py` / `tests/test_web_tools.py`：

```python
def test_word_count_basic(tmp_path):
    from src.tools.contrib.example_tools import word_count
    result = word_count("hello world\n", base_dir=str(tmp_path))
    assert result.success is True
    assert "字符数" in result.output
```

关键点：
- 用 `tmp_path` 隔离文件系统；
- 断言 `result.success` 与 `result.output` / `result.error`；
- 网络工具必须 mock，不访问外网。

## 八、检查清单

新增工具前确认：
- [ ] `name` 全局唯一，语义清晰
- [ ] `description` 写清用途（模型靠它决定何时调用）
- [ ] `params` 的 schema 准确，可选参数有 `default`
- [ ] 函数内 `try/except`，返回 `ToolResult` 而非抛异常
- [ ] `category` 正确（只读工具别标 `write`）
- [ ] 路径类工具用 `resolve_path` 防越界
- [ ] 在 `worker_tools.py` 顶部 import 触发注册
- [ ] 补测试，运行 `python -m pytest -q` 全绿
