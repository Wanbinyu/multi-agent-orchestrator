## Phase 7：证据驱动工程 Agent

### 目标

为 MAO 增加统一的工程决策层：任务分类、项目侦察、计划状态、假设与证据、风险分级验证、完成审计以及基于证据的 Reviewer。

### 启动条件

- Phase 6.6 P0：CLI 最终回答可见（已满足）。
- Phase 6.6 P1：`project_tree` 与 `/tree` 可用（已满足）。

### 计划文档

- `docs/Phase7-证据驱动工程Agent计划.md`
- `docs/开源发布准备计划.md`

### 当前实现

- 每个同步、流式和协作回合创建独立 `RunJournal`，原子写入 `sessions/<session_id>/runs/<run_id>.yaml`。
- 已定义任务意图、工作计划、证据、验证门和运行日志模型，并约束工作计划状态转换。
- CLI 显示运行 ID 与结果；Web 显示“本轮记录”，SSE 提供工程开始、更新和完成事件。
- 提供会话运行记录列表与详情 API；正常完成和受控异常分别落为 `completed`、`failed`。
- Web 失败流在 `finally` 中保存会话，模型请求失败后刷新页面仍保留用户消息。
- 真实 `glm-ark` 401 鉴权失败路径已验证，运行记录为 `failed`，浏览器控制台无错误。
- Phase 7.1 确定性任务分类已接入：问答、解释、诊断、修改、构建、审查、方案和监控具有独立策略。
- 只读任务在 auto 模式下仍禁止项目写入；change/build 才能按 approve/auto 边界获得写权限。
- CLI/Web 工程记录显示任务类型、风险和“只读 / 写入需批准 / 写入已授权”。
- Phase 7.2 将真实工具结果自动转换为可去重 Evidence；失败证据同时保留工具输出和错误原因，不从模型正文伪造证据。
- 项目侦察跟踪结构、Git、文档、依赖、入口和测试六类覆盖；缓存命中不重复计数，跳过读取不会冒充已检查。
- 新增固定参数的只读 `git_status`；同步和流式工具执行后立即原子保存证据，流式路径发送 `engineering_update`。
- `Hypothesis` 支持未测试、支持、反证和不确定状态，支持/反证必须绑定已存在证据。
- CLI/Web 本轮记录显示证据条数和侦察覆盖；完整证据详情仍按计划放在 Phase 7.5。
- Phase 7.3 风险验证策略已接入：标准修改要求 targeted + adjacent，高风险构建要求 targeted + integration + full + smoke。
- 需求矩阵绑定实现 Evidence 与 VerificationGate；build 的使用说明必须有文档写入证据。
- 完成审计会把缺少直接证据的 `completed` 降为 `blocked`，并同步改写最终答复，禁止无验证完成声明。
- Reviewer 消费确定性审计上下文，模型输出和非 JSON 兜底均不能绕过失败审计。
- CLI/Web 显示验证门数量、审计状态和具体缺口。
