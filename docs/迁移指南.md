# MAO 迁移与升级指南

本指南汇总 `v0.1.0-beta.3` 至 `v0.1.0-beta.6` 之间需要用户注意的变更，以及向 `v0.2.0` 迁移的要点。每个版本的完整变更见对应的 `docs/RELEASE_NOTES_v0.1.0-beta.*.md`。

MAO 在 `0.x` 阶段可能包含破坏性变更，会在 Release Notes 与本指南显式标注。升级前请备份本地 `config/`、`sessions/`、`memory/`（这些不入库）。

## 升级

```bash
pipx upgrade multi-agent-orchestrator
# 或源码
git pull --ff-only
python -m pip install -e ".[test]"
python -m pytest -q
```

升级后首次运行 `mao` 会按需迁移本地配置；旧 `providers.yaml`/`workers.yaml` 保持兼容。

## beta.3 -> beta.4

- **权限模式双重门控**：`auto` 可直接写入/执行；`approve` 只确认非只读工具；`readonly` 可读取但拒绝写入/命令。未分类请求跟随会话模式，不再误报为工程变更。句中"帮我做/把……搭建好"识别为 build。
- **会话恢复**：最新 run 为 `running`/`blocked`/未完成计划时，CLI `/resume` 与 Web 横幅在显式继续或放弃前阻断新消息。继续只把未完成步骤检查点交给新 run，不自动重放已完成工作。
- **分层压缩**：L0 artifact 引用 + L1 结构化摘要 + L2 近期全文；原生 `tool_use`/`tool_result` 不被拆开。
- **Reviewer 默认 restricted**：只读需求/计划/文件/直接证据，不看 Worker 正文；`workers.yaml` 可切 `full`。
- **持久 Plan 模式**：CLI/Web 可进入、修订、批准实施、取消。

## beta.4 -> beta.5

- **执行深度**：`/depth auto|fast|standard|deep`；高风险任务不能用 `fast` 绕过确定性验证。
- **可解释路由**：`/routing fixed` 锁定主模型；自动路由最多一次升级，未验证能力不升级，未知价格不宣称节省。
- **模型目录单一真值**：CLI 与 Web 预设统一由 `src/models/catalog.py` 生成；`ark` Coding Plan 预设改为从 catalog 取值（重新生成的配置可能多出 capability/metadata 字段）。
- **对抗测试**：默认关闭；`/adversarial on|off` 仅在显式启用的 `deep change/build` 协作中运行只读 `AdversarialTester`。
- **基准**：`python scripts/benchmark_engineering.py` 离线运行，结果为 `synthetic_contract`，不代表真实模型效果。

## beta.5 -> beta.6

- **Plugin API v0**：插件默认关闭，`config/plugins.yaml` 显式启用（已加入 `.gitignore`）。
- **`contrib/example_tools.py` 模式**：导入即注册的旧模式仍可用，但分发用插件应改用 Plugin API（manifest + entry point + 启用门控）。见 [`插件开发指南.md`](插件开发指南.md)。
- **`mao plugin` CLI**：新增 `list/doctor/enable/disable` 子命令。
- **Web**：新增 `GET /api/plugins` 与 chat「插件」只读标签。
- **隔离**：`ToolRegistry`/`HookRegistry` 新增注销方法，支持插件加载失败回滚。

## 向 v0.2.0 迁移

`v0.2.0` 不自动发布。进入条件见 [`版本计划-v0.1.0-beta.3至beta.6.md`](版本计划-v0.1.0-beta.3至beta.6.md) §6。迁移要点：

- **Provider 兼容矩阵**：升级后核对所用 Provider 的能力状态（`unverified`/`supported`）；未验证能力不参与自动路由升级。
- **Plugin API 兼容**：`0.x` 阶段 Plugin API 仍可能破坏性演进；插件作者应锁定 `mao_api_version` 并关注 Release Notes。兼容规则见 [`Plugin-API兼容策略.md`](Plugin-API兼容策略.md)。
- **真实基准**：B5.4 真实多模型对比需所有者重新授权累计次数边界后恢复；在此之前不得用 MAO 的基准数据宣传真实模型优势。
- **配置位置不变**：`config/`、`sessions/`、`memory/`、`.env` 仍为本地，不入库。

## 回滚

如升级后出现问题，可固定到上一版本：

```bash
pipx install git+https://github.com/Wanbinyu/multi-agent-orchestrator.git@v0.1.0-beta.5
```

本地 `config/`、`sessions/` 向前兼容；如遇不兼容的 RunJournal 版本，MAO 会以只读方式加载旧记录并在新 run 中使用当前格式。
